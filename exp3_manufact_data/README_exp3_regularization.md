## 题目 1：机器学习正则化中的范数应用——Exp3 数值实验说明（制造业合成数据）

### 一、实验目的与整体设计

- **目的**：通过一个“高维、少数有效特征”的合成制造业场景，对比 L1（Lasso）与 L2（Ridge）正则化在系数结构、预测性能和数值稳定性上的差异，并与“无正则化”基线比较，为 PPT 中的
  \[
  J_1(\beta)=\frac{1}{2m}\lVert X\beta-y\rVert_2^2+\lambda\lVert\beta\rVert_1,\quad
  J_2(\beta)=\frac{1}{2m}\lVert X\beta-y\rVert_2^2+\lambda\lVert\beta\rVert_2^2
  \]
  提供直观数值例子。
- **脚本位置**：`exp3_manufact_data/test3.py`。
- **输出图片**：运行脚本后，在 `exp3_manufact_data` 下生成
  `fig1_mse_vs_lambda.png` ~ `fig6_pred_vs_true.png` 六张图。

### 二、数据与模型设定

- **数据生成**（函数 `generate_synthetic_manufact_data`）：
  - 样本数 `N_SAMPLES = 150`，特征数 `N_FEATURES = 200`。
  - 只有前 `N_INFORMATIVE = 10` 个特征是真正有用的，真实系数 `beta_true` 仅在这 10 维为非零，其余 190 维为 0。
  - 噪声：在线性模型 \(y = X\beta_{\text{true}} + \varepsilon\) 中加入高斯噪声，方差由 `NOISE_STD = 0.5` 控制。
- **数据拆分与标准化**：
  - 使用 `train_test_split` 按 7:3 划分训练集 / 测试集。
  - 对特征做 `StandardScaler` 标准化（训练集 `fit_transform`，测试集 `transform`），保证各特征在同一尺度上，方便比较正则化效果。
- **三种模型**：
  - **无正则化**：`LinearRegression`。
  - **L1 正则（Lasso）**：`Lasso(alpha=λ)`，惩罚项是 \(\lambda\lVert\beta\rVert_1\)。
  - **L2 正则（Ridge）**：`Ridge(alpha=λ)`，惩罚项是 \(\lambda\lVert\beta\rVert_2^2\)。

### 三、λ 扫描与“最佳 λ”选择

- 在一组预设的 `LAMBDA_LIST = np.linspace(0.001, 5, 30)` 上：
  - 对每个 λ，分别训练 Lasso 和 Ridge，在测试集上计算 MSE：
    \[
    \text{MSE}(\lambda) = \frac{1}{m_\text{test}}\sum_i (y_i-\hat y_i(\lambda))^2.
    \]
  - 记录每个 λ 的测试 MSE 和对应系数向量。
- 通过 `np.argmin` 找到使测试 MSE 最小的 λ，记为：
  - `lambda_best_lasso`、`lambda_best_ridge`。
- 使用这两个最佳 λ 重新训练 Lasso 和 Ridge，得到“最终模型”系数与预测，用于后续图 4~图 6 以及数值分析。
- 另外使用一个更密的 `LAMBDA_PATH = np.logspace(-3, 1, 50)` 重新训练 Lasso/Ridge，得到**系数路径**（每个特征系数随 λ 变化的曲线），用来体现 L1 与 L2 正则化的结构差异。

### 四、为何 Lasso 拟合效果更好？

在当前这类“高维、少数有效特征”的设定下，实验结果典型地表现为：

- **Lasso 的最佳 λ 较小（例如 ≈0.17），非零系数个数接近真实有用特征个数（10 个）**：
  - L1 惩罚鼓励系数精确为 0，从 200 个特征中自动筛出了少量真正有贡献的特征；
  - 这相当于做了**特征选择**，降低了方差、缓解了噪声与无关特征带来的过拟合。
- **Ridge 的所有 200 个系数几乎都非零**：
  - L2 惩罚只是“缩小”系数，不倾向于把它们压到 0；
  - 在本实验中，由于有 190 个真实系数为 0，Ridge 会把这些维度的系数也当成需要拟合的参数，引入多余自由度。
- 在测试集上：
  - **Lasso 的预测 MSE 明显小于 Ridge 和无正则化**，同时系数恢复误差 \(\text{MSE}(\hat\beta, \beta_{\text{true}})\) 也更接近 0；
  - 这说明在“稀疏真相 + 噪声”的设定下，**L1 正则更适合恢复稀疏结构**，在有限样本下能取得更好的泛化效果。
- 理论上，这与 L1 正则的**尖角几何形状**有关：
  - L1 球在坐标轴方向有尖角，最优解更容易落在坐标轴上（某些坐标为 0），从而产生稀疏解；
  - L2 球是圆形，最优解一般不会恰好落在坐标轴上，因此很少出现精确 0 的系数。

综上，在这个“少量重要特征 + 大量冗余特征”的制造业合成场景中，Lasso 通过稀疏化显著优于 Ridge。

### 五、各张图表的说明

#### 图 1：`fig1_mse_vs_lambda.png` — 测试 MSE vs λ

- **横轴**：正则化强度 λ（对数坐标）。
- **纵轴**：测试集 MSE。
- **三条曲线/线段**：
  - 蓝色：Lasso 在不同 λ 下的测试 MSE。
  - 红色：Ridge 在不同 λ 下的测试 MSE。
  - 灰色虚线：无正则化模型的测试 MSE（一个常数基线）。
- **解读**：
  - λ 太小时，正则化几乎不起作用，Lasso/Ridge 接近无正则化，容易过拟合，MSE 偏大；
  - λ 适中时，Lasso 的 MSE 降到最低（最佳 λ），说明适度 L1 惩罚提高了泛化能力；
  - λ 太大时，两个模型都过度收缩，欠拟合导致 MSE 上升。

#### 图 2：`fig2_lasso_coef_path.png` — Lasso 系数路径

- **横轴**：λ（对数坐标）。
- **纵轴**：系数数值。
- **曲线**：前 `N_SHOW` 个特征（默认 10 个）在 `LAMBDA_PATH` 上的系数变化。
- **解读**：
  - 当 λ 较小时，多条曲线都非零；随着 λ 增大，许多曲线逐渐被压到 0；
  - 特别是那些对应“无用特征”的曲线更早、更多地变为 0；
  - 这清楚展示了 L1 正则的**稀疏化效果**：通过惩罚 \(\lVert\beta\rVert_1\) 把一部分系数直接消掉。

#### 图 3：`fig3_ridge_coef_path.png` — Ridge 系数路径

- **横轴**：λ（对数坐标），**纵轴**：系数数值。
- **曲线**：同样是前 `N_SHOW` 个特征的系数路径。
- **解读**：
  - 随着 λ 增大，所有系数的绝对值都在逐渐减小，但很少精确为 0；
  - 各条曲线相比 Lasso 更平滑、变化更温和；
  - 这体现出 L2 正则的特点：**整体缩小系数、提高数值稳定性，但不做显式特征选择**。

#### 图 4：`fig4_coef_compare.png` — 最佳 λ 下三种方法的系数对比

- **横轴**：前 `N_SHOW` 个特征索引。
- **纵轴**：系数数值。
- **三组柱状**：
  - `No reg`：无正则化的系数；
  - `Lasso`：最佳 λ 下 Lasso 的系数；
  - `Ridge`：最佳 λ 下 Ridge 的系数。
- **解读**：
  - 可直观看到：
    - Lasso 只保留少数非零系数，其位置与 `beta_true` 非零位置基本一致；
    - Ridge 对所有特征都给出非零系数，且相比无正则化被整体缩小；
  - 这张图直接对应 PPT 中“L1 与 L2 惩罚项对解结构的影响”的讨论，可以用来说明：
    - L1：产生稀疏解，做特征选择；
    - L2：所有特征都被保留，只是幅度被压小，偏向更平滑、更稳定。

#### 图 5：`fig5_residual_boxplot.png` — 残差分布箱线图

- **横轴**：三种模型（No reg / Lasso / Ridge）。
- **纵轴**：测试集残差 \(y_{\text{test}}-\hat y\)。
- **内容**：每种模型的残差箱线图（中位数、四分位数、异常值）。
- **解读**：
  - 通过箱体高度和须长，可以比较三种方法的**误差分布与稳定性**；
  - 通常情况下，Lasso 的箱体会比无正则化更集中，说明整体误差较小；
  - Ridge 的残差分布有时比无正则化稍好，但在稀疏真相场景下仍不如 Lasso。

#### 图 6：`fig6_pred_vs_true.png` — 预测值 vs 真实值散点图

- **横轴**：真实值 \(y_{\text{test}}\)。
- **纵轴**：预测值 \(\hat y\)。
- **散点**：三种模型的预测点，颜色区分；虚线为理想的 \(y=x\) 直线。
- **解读**：
  - 点越贴近对角线，预测越准确；
  - 实验中可以看到 Lasso 的点云更靠近 \(y=x\)，而无正则化和 Ridge 的点云更散；
  - 这张图直观展示了正则化对预测性能和数值稳定性的影响。

### 六、脚本流程小结

1. 生成带稀疏真实系数的高维合成数据 `(X, y, beta_true)`。
2. 划分训练 / 测试集，对特征做标准化。
3. 训练无正则化线性回归，得到基线预测与 MSE。
4. 在一系列 λ 上分别训练 Lasso 与 Ridge，记录测试 MSE，选出各自最佳 λ。
5. 用最佳 λ 重新训练 Lasso/Ridge，得到最终系数、预测与残差。
6. 在细致的 `LAMBDA_PATH` 上再次训练，得到系数路径，用于图 2 和图 3。
7. 绘制并保存 6 张图，使 L1 与 L2 在 MSE、系数结构、残差分布和预测表现上的差异一目了然。
8. 在控制台打印最佳 λ、非零系数个数、范数和系数恢复误差，并给出简要中文分析，方便与 PPT 中的四个“解题角度”对应。

总体而言，这个脚本给出了一个完整的“从范数定义到数值实验”的闭环示例：  
从 L1/L2 范数的几何性质出发，通过合成稀疏真相的数据构造、三种模型训练、λ 扫描与多视角可视化，清晰展示了 Lasso 与 Ridge 在**稀疏性、泛化能力与数值稳定性**上的差异，可直接作为课堂或报告中的案例支撑。
